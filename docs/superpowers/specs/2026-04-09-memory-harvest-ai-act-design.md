# Memory Harvest + AI Act Compliance — Design Spec

**Date:** 2026-04-09
**Status:** Approved
**Author:** Marko Markovic + Claude
**Phases:** ULTRAPLAN 7a (Memory Harvest) + 7b (AI Act Compliance)

---

## 1. Problem Statement

Waggle's strategic weapon is persistent memory, but users build AI knowledge across 20+ platforms (ChatGPT, Claude, Gemini, Perplexity, Grok, Cursor, Claude Code, Manus, Genspark, Qwen, Minimax, z.ai, etc.). That knowledge is siloed and invisible.

Meanwhile, the EU AI Act (full application Aug 2, 2026) requires automatic interaction logging, human oversight tracking, 6-month retention, and risk classification for high-risk AI deployments. No product combines being the AI workspace AND governing it.

**Memory Harvest** consolidates all AI conversations into Waggle's governed memory.
**AI Act Compliance** turns Waggle's audit infrastructure into a regulatory-ready compliance layer.

Together: "You can't audit what you can't see. Waggle sees everything because it harvests everything."

---

## 2. Architecture Overview

```
[20+ AI Tools] → Source Adapters → Universal Import Items → 4-Pass LLM Pipeline → Waggle Memory
                                                                  │
                                                        ├── Pass 1: Classify (Haiku)
                                                        ├── Pass 2: Extract (Sonnet)
                                                        ├── Pass 3: Synthesize (Sonnet)
                                                        └── Pass 4: Dedup (local, no LLM)
                                                                  │
                                                                  ▼
                                              ┌─── Identity Layer (personal facts, prefs)
                                              ├─── I-Frames (decisions, knowledge, facts)
                                              ├─── KG Entities + Relations
                                              └─── Provenance Chain (AI Act Art. 12)
```

## 3. Universal Import Format

### 3.1 Import Item (adapter output)

```typescript
type ImportSourceType =
  | 'chatgpt' | 'claude' | 'claude-code' | 'claude-desktop'
  | 'gemini' | 'google-ai-studio' | 'perplexity' | 'grok'
  | 'cursor' | 'copilot' | 'manus' | 'genspark'
  | 'qwen' | 'minimax' | 'z-ai' | 'openclaw' | 'cowork'
  | 'elevenlabs' | 'google-flow' | 'unknown';

type ImportItemType =
  | 'conversation' | 'memory' | 'instruction'
  | 'preference' | 'artifact' | 'rule';

interface UniversalImportItem {
  id: string;                          // UUID, generated by adapter
  source: ImportSourceType;
  type: ImportItemType;
  title: string;
  content: string;                     // Raw text content
  messages?: ConversationMessage[];    // If type=conversation
  timestamp: string;                   // ISO 8601
  metadata: Record<string, unknown>;   // Source-specific data preserved
}
```

### 3.2 Distilled Knowledge (pipeline output)

```typescript
interface DistilledKnowledge {
  targetLayer: 'identity' | 'frame' | 'kg_entity' | 'kg_relation' | 'awareness';
  frameType?: 'I' | 'P';
  importance: Importance;              // critical | important | normal | temporary
  source: 'import';                    // Always 'import' provenance
  content: string;
  entities?: { name: string; type: string }[];
  relations?: { source: string; target: string; relation: string }[];
  provenance: {
    originalSource: ImportSourceType;
    originalId?: string;               // Conversation ID from source platform
    conversationTitle?: string;
    importedAt: string;
    distillationModel: string;         // Which LLM did the extraction
    confidence: number;                // 0-1
    pass: number;                      // Which pipeline pass produced this
  };
}
```

## 4. Source Adapters

### 4.1 Tier 1 — Structured Adapters (known formats)

| Adapter | Source | Input | Method |
|---------|--------|-------|--------|
| `chatgpt-adapter.ts` | ChatGPT | JSON export (Settings → Export) | Parse `mapping` node tree (exists, upgrade) |
| `claude-adapter.ts` | Claude web/desktop | JSON export | Parse `chat_messages` array (exists, upgrade) |
| `claude-code-adapter.ts` | Claude Code | Filesystem `~/.claude/` | Read memory/*.md, rules/, plans/, sessions |
| `gemini-adapter.ts` | Gemini / AI Studio | Google Takeout JSON | Parse conversation array |
| `cursor-adapter.ts` | Cursor IDE | Local SQLite | Read conversation history DB |

### 4.2 Tier 2 — Universal Adapter (LLM-assisted)

`universal-adapter.ts` — accepts any text, JSON, Markdown, or HTML. LLM auto-detects:
- Source platform (from formatting cues, metadata)
- Content structure (conversations, settings, notes)
- Produces `UniversalImportItem[]`

Handles all 20 tools immediately via paste/upload. User drops whatever they can export/copy.

### 4.3 Claude Code Filesystem Harvester (special case)

Reads directly from disk — no export needed:

```
~/.claude/
├── projects/<hash>/memory/
│   ├── MEMORY.md           → Parse index, extract descriptions
│   └── *.md                → Parse frontmatter (type, name, description)
│                              type: user     → Identity layer
│                              type: feedback → Preference I-frames (important)
│                              type: project  → Workspace-scoped frames
│                              type: reference → KG entities
├── rules/**/*.md            → Import as behavioral preferences
├── plans/*.md               → Import as project context
└── settings.json            → Extract model preferences, tool config
```

Default path: `~/.claude/` (auto-detected). User can override in Settings.

## 5. 4-Pass Distillation Pipeline

### 5.1 Pass 1: Classify (Haiku — cheap)

Input: `UniversalImportItem[]` (batch of 20)
Output: Each item tagged with:
- `domain`: work | personal | technical | mixed
- `value`: high | medium | low | skip
- `categories`: string[] (e.g., ['decision', 'architecture', 'preference'])

Items tagged `skip` are dropped (greetings, trivial exchanges, repetitive debugging loops).

### 5.2 Pass 2: Extract (Sonnet — accurate)

Input: Classified items with value >= medium
Output: Per item:
- `decisions`: string[] — choices made, approaches selected
- `preferences`: string[] — user likes/dislikes, working style
- `facts`: string[] — personal info, role, stack, company
- `knowledge`: string[] — technical insights, research findings
- `entities`: { name, type }[] — people, projects, technologies, organizations
- `relations`: { source, target, relation }[] — entity relationships

### 5.3 Pass 3: Synthesize (Sonnet — accurate)

Input: Extractions from Pass 2
Output: `DistilledKnowledge[]` — ready to write to Waggle memory:
- Identity updates (merge with existing identity layer)
- I-frames with proper importance levels
- KG entities with type classification
- KG relations with confidence scores
- Full provenance chain on every item

### 5.4 Pass 4: Dedup (local — no LLM)

Input: `DistilledKnowledge[]` + existing Waggle memory
Operations:
- Content-hash dedup against existing frames
- Entity name normalization (fuzzy match existing KG entities)
- Conflict detection (new fact contradicts existing frame → flag for user review)
- Merge statistics

### 5.5 Cost Estimate

For 1,000 conversations (~500K input tokens):
- Pass 1 (Haiku): ~$0.12
- Pass 2 (Sonnet): ~$1.50
- Pass 3 (Sonnet): ~$1.00
- Pass 4 (local): $0
- **Total: ~$2.60**

## 6. Incremental Sync

### 6.1 Harvest Sources Table

```sql
CREATE TABLE IF NOT EXISTS harvest_sources (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  source TEXT NOT NULL UNIQUE,
  display_name TEXT NOT NULL,
  source_path TEXT,                      -- filesystem path for local sources
  last_synced_at TEXT,
  items_imported INTEGER NOT NULL DEFAULT 0,
  frames_created INTEGER NOT NULL DEFAULT 0,
  auto_sync INTEGER NOT NULL DEFAULT 0,  -- 1 = cron enabled
  sync_interval_hours INTEGER NOT NULL DEFAULT 24,
  last_content_hash TEXT,
  created_at TEXT NOT NULL DEFAULT (datetime('now'))
);
```

### 6.2 Sync Modes

**Local sources (Claude Code, Cursor):**
- Background cron job via existing `cron-store.ts`
- Compares file mtimes against `last_synced_at`
- Only processes changed/new files
- Configurable interval (default: 24h)

**Web sources (ChatGPT, Gemini, etc.):**
- Manual re-import with dedup
- Smart reminder notifications after N days (default: 14)
- Track processed conversation IDs to skip on re-import

### 6.3 Notification Flow

When cron detects new data or a source is stale:
- Add entry to NotificationInbox: "12 new Claude Code sessions since last harvest"
- StatusBar shows harvest dot indicator
- Clicking opens MemoryApp → Harvest tab

## 7. AI Act Compliance Layer

### 7.1 Interaction Audit Table (Art. 12)

```sql
CREATE TABLE IF NOT EXISTS ai_interactions (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  timestamp TEXT NOT NULL DEFAULT (datetime('now')),
  workspace_id TEXT,
  session_id TEXT,
  model TEXT NOT NULL,
  provider TEXT NOT NULL,
  input_tokens INTEGER NOT NULL DEFAULT 0,
  output_tokens INTEGER NOT NULL DEFAULT 0,
  cost_usd REAL NOT NULL DEFAULT 0,
  tools_called TEXT NOT NULL DEFAULT '[]',
  human_action TEXT CHECK (human_action IN ('approved', 'denied', 'modified', 'none')),
  risk_context TEXT,
  imported_from TEXT,
  persona TEXT
);
CREATE INDEX IF NOT EXISTS idx_interactions_workspace ON ai_interactions (workspace_id, timestamp);
CREATE INDEX IF NOT EXISTS idx_interactions_timestamp ON ai_interactions (timestamp DESC);
CREATE INDEX IF NOT EXISTS idx_interactions_model ON ai_interactions (model);
```

### 7.2 Workspace Risk Classification (Art. 26)

Add to workspace config:

```typescript
type AIActRiskLevel = 'minimal' | 'limited' | 'high-risk' | 'unacceptable';
```

Auto-suggested from template:
- HR/Recruiting → `high-risk` (Cat 4: Employment)
- Legal → `high-risk` (Cat 8)
- Finance → `high-risk` (Cat 5: Credit/Insurance)
- Sales/Marketing/Research → `limited`
- Code/General → `minimal`

User can override. Classification change logged to `ai_interactions`.

### 7.3 Compliance Status Check

Function that evaluates compliance per workspace:

```typescript
interface ComplianceStatus {
  art12_logging: { status: 'compliant' | 'warning' | 'non-compliant'; totalInteractions: number; coverage: string };
  art14_oversight: { status: string; humanActions: number; approvalRate: string };
  art19_retention: { status: string; oldestLog: string; retentionDays: number };
  art26_monitoring: { status: string; activeMonitors: string[] };
  art50_transparency: { status: string; modelsDisclosed: boolean };
}
```

### 7.4 Worker Notification (Art. 26(7))

First-run disclosure per workspace:
> "This workspace uses AI (model: [X]) for [template purpose]. Your interactions are logged per EU AI Act Art. 12. [Acknowledge]"

Stored in workspace config as `ai_act_acknowledged: boolean`.

### 7.5 FRIA Template (Art. 27)

For high-risk workspaces, generate Fundamental Rights Impact Assessment pre-filled with:
- Workspace metadata (purpose, template, personas)
- Tools available and capabilities
- Data sources (imported + live)
- Risk mitigation measures (approval gates, deny lists, audit logging)
- Human oversight assignments

Export as structured JSON (DOCX generation deferred to existing document tooling).

## 8. Audit Report

### 8.1 Export Format

**JSON** — machine-readable, full data:
- Report metadata (version, period, org, generator)
- Workspace metadata + risk classification
- Compliance status per article (12, 14, 19, 26, 50)
- Model inventory (models used, call counts, token counts, costs)
- Human oversight log (every approve/deny/modify with timestamp)
- Harvest provenance (sources, import dates, item counts)
- Full interaction log (paginated)

**PDF** — human-readable formal report:
- Executive summary with compliance checkmarks per article
- Model inventory table
- Human oversight summary (approved/denied/modified percentages)
- Data provenance summary
- Risk assessment section
- Reference to full interaction log as appendix

### 8.2 API Endpoint

```
POST /api/compliance/export
Body: {
  workspaceId?: string,    // null = all workspaces
  from: string,            // ISO date
  to: string,              // ISO date
  format: 'json' | 'pdf' | 'both',
  include: {
    interactions: boolean,
    oversight: boolean,
    models: boolean,
    provenance: boolean,
    riskAssessment: boolean,
    fria: boolean
  }
}
Response: { json?: object, pdfUrl?: string }
```

## 9. UI Design

### 9.1 Harvest Tab (MemoryApp — third view)

MemoryApp tabs: `[Timeline] [Graph] [Harvest]`

Harvest tab shows:
- **Source cards** — one per connected source, showing: icon, name, last sync, items imported, auto-sync toggle
- **Auto-detect banner** — if `~/.claude/` found: "Claude Code detected — 47 memories, 12 plans"
- **Add Source** — opens source selector (Tier 1 structured sources + universal paste/upload)
- **Import progress** — streaming progress bar during distillation pipeline

### 9.2 Onboarding Integration

Step 3 (MemoryImport) enhanced:
- Auto-detect local sources (Claude Code, Cursor)
- Multi-source upload for ChatGPT/Claude/Gemini JSON
- Universal paste for everything else
- Preview before commit with item counts by category
- This is the "wow moment" — Waggle already knows you

### 9.3 Compliance Dashboard (CockpitApp card)

Real-time compliance monitoring card:
- Overall status badge (COMPLIANT / WARNING / NON-COMPLIANT)
- Per-article status indicators (Art. 12, 14, 19, 26, 50)
- Per-workspace risk badges
- Last interaction timestamp + oldest log date
- [Export Report] button → opens report generator

### 9.4 Report Generator

Accessible from: CockpitApp compliance card → Export, or Settings → Compliance tab
- Workspace selector (all or specific)
- Date range picker
- Checkboxes for report sections
- Format toggle (JSON / PDF / Both)
- Recent reports list with download links

### 9.5 StatusBar Harvest Indicator

Dot indicator next to notifications when unharvested data exists.
Click opens MemoryApp → Harvest tab.

### 9.6 Workspace Risk Badges

On DashboardApp workspace cards:
- Risk level badge (color-coded: green=minimal, blue=limited, amber=high-risk, red=unacceptable)
- Mini compliance indicators: "Art.12 ✓ Art.14 ✓ Art.19 ✓"
